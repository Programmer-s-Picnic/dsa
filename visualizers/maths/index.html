Addition as joining piles

Addition is the most concrete pebble action: take one pile, add another pile, and count the total. can also show the merge in stages.

Strongest idea: addition means combining quantities. Step mode shows the story in three parts: first pile, second pile, then total.

Subtraction as removing pebbles

This section now separates two learning modes: +ve only, where the visual model stops at zero, and +ve and -ve, where extra removal is shown using negative pebbles.

+ve only is best for beginners. +ve and -ve helps bridge toward integers and negative results.

Multiplication as repeated groups

Multiplication is shown both as rows and as repeated addition. This makes the array model and the arithmetic model support each other.

Key idea: a × b can be read as b rows of a, or a added to itself b times.

Division as equal grouping

Division now emphasizes the inverse link with multiplication and highlights the remainder more clearly.

The quotient counts full groups. The remainder is what cannot be packed into one more full group.

Factorial as growing arrangements

This builds 1!, 2!, 3!, and 4! by inserting the new pebble into every slot. Each stage now includes clearer summaries.

Interpretation: n! is the number of ways to arrange n distinct things in order.

Permutations as ordered arrangements

This now explicitly states the difference from combinations: in permutations, order matters.

input help: type symbols or comma-separated labels. Example: AABC gives A×2, B×1, C×1.
Example: 1,2,3 and 3,2,1 are different permutations because the order changed. If some elements repeat, only unique orders are counted.

Combinations as choosing without order

This version makes the no-order rule more explicit and adds the factorial formula directly to the explanation.

input help: order will be ignored here, but available copies still matter.
If order does not matter, {1,4,5} and {5,1,4} count as the same choice. With repeated elements, only unique selections allowed by available counts are kept.

Powers as repeated choice or repeated multiplication

This connects exponentiation to both choice trees and repeated multiplication so the symbol a^b feels less mysterious.

Interpret it in two ways: a multiplied by itself b times, or b positions each with a choices.
