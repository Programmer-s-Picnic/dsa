Lesson Goal

OOPs means Object-Oriented Programming System. It is a way of writing programs by grouping related data and related actions together.

In simple words: an object has data and can perform actions. A student has a name and marks. A bank account has a balance and can deposit or withdraw money.

1. Why do we need OOPs?

Suppose we are creating a program for students. Without OOPs, we may write many separate variables. 

student1_name = "Ravi"
student1_marks = 85

student2_name = "Raveena"
student2_marks = 92

def show_student(name, marks):
    print(name, marks)

show_student(student1_name, student1_marks)
show_student(student2_name, student2_marks)

This works, but as the program grows, it becomes difficult to manage. OOPs allows us to create proper student objects. 

class Student:
    def show(self):
        print(self.name, self.marks)

s1 = Student()
s1.name = "Ravi"
s1.marks = 85

s2 = Student()
s2.name = "Raveena"
s2.marks = 92

s1.show()
s2.show()

2. Class and Object

Class A class is a blueprint. It defines what kind of data and actions an object will have.
Object An object is a real item created from a class.

Attributes

Attributes are data stored inside an object.

class Student:
    pass

print(s1.name)

Methods

Methods are functions written inside a class.

class Student:

s1 = Student()
s1.show()

3. The self Keyword and Constructor

What is self?

self means the current object. It allows a method to use the data of the object that called it. 

class Student:
    def show(self):
        print(self.name, self.marks)

s1 = Student()
s1.name = "Ravi"
s1.marks = 85

s1.show()

What is __init__()?

__init__() is a constructor. It runs automatically when a new object is created. 

class Student:
    def __init__(self, name, marks):
        self.name = name
        self.marks = marks

    def show(self):
        print(self.name, self.marks)

s1 = Student("Ravi", 85)
s2 = Student("Raveena", 92)

s1.show()
s2.show()

4. Four Pillars of OOPs

Pillar Meaning
Encapsulation Keeping data and methods together inside a class.
Abstraction Showing only what is needed and hiding internal details.
Inheritance Creating a new class from an existing class.
Polymorphism Same name, different behavior.

Encapsulation Example

class Student:
    def __init__(self, name, marks):
        self.name = name
        self.__marks = marks

    def get_marks(self):
        return self.__marks

    def set_marks(self, marks):
        if marks >= 0 and marks <= 100:
            self.__marks = marks
        else:
            print("Invalid marks")

s1 = Student("Ravi", 85)

print(s1.get_marks())
s1.set_marks(95)
print(s1.get_marks())

Abstraction Example

class Remote:
    def press_power_button(self):
        self.__connect_circuit()
        print("TV turned on")

    def __connect_circuit(self):
        print("Internal circuit connected")

remote = Remote()
remote.press_power_button()

5. Inheritance

Inheritance means creating a new class from an existing class. The existing class is the parent class. The new class is the child class. 

class Person:
    def __init__(self, name):
        self.name = name

    def show_name(self):
        print("Name:", self.name)

class Student(Person):
    def __init__(self, name, marks):
        super().__init__(name)
        self.marks = marks

    def show_marks(self):
        print("Marks:", self.marks)

s1 = Student("Ravi", 85)

s1.show_name()
s1.show_marks()

Types of Inheritance

# Single inheritance
class A:
    pass

class B(A):
    pass


# Multilevel inheritance
class A:
    pass

class B(A):
    pass

class C(B):
    pass


# Multiple inheritance
class Father:
    def skill1(self):
        print("Driving")

class Mother:
    def skill2(self):
        print("Cooking")

class Child(Father, Mother):
    pass

c = Child()
c.skill1()
c.skill2()

6. Polymorphism

Polymorphism means one name having many forms. 

print(len("Python"))
print(len([10, 20, 30]))
print(len({"a": 1, "b": 2}))

Method Overriding

class Animal:
    def speak(self):
        print("Animal makes sound")

class Human(Animal):
    def speak(self):
        print("Human speaks language")

h = Human()
h.speak()

Operator Overloading

class Number:
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return self.value + other.value

n1 = Number(10)
n2 = Number(20)

print(n1 + n2)

7. Composition

Composition means one object contains another object. It represents a has-a relationship. 

class Engine:
    def start(self):
        print("Engine started")

class Car:
    def __init__(self):
        self.engine = Engine()

    def start_car(self):
        self.engine.start()
        print("Car started")

car = Car()
car.start_car()
Inheritance means is-a. Composition means has-a. A student is a person. A car has an engine.

8. Complete Mini Project: Student Result System

class Student:
    def __init__(self, roll, name, marks):
        self.roll = roll
        self.name = name
        self.marks = marks

    def total(self):
        return sum(self.marks)

    def percentage(self):
        return self.total() / len(self.marks)

    def grade(self):
        percent = self.percentage()

        if percent >= 90:
            return "A"
        elif percent >= 75:
            return "B"
        elif percent >= 60:
            return "C"
        elif percent >= 40:
            return "D"
        else:
            return "Fail"

    def show_report(self):
        print("Roll:", self.roll)
        print("Name:", self.name)
        print("Marks:", self.marks)
        print("Total:", self.total())
        print("Percentage:", self.percentage())
        print("Grade:", self.grade())
        print("----------------------")

s1 = Student(1, "Ravi", [80, 85, 90])
s2 = Student(2, "Raveena", [95, 92, 96])
s3 = Student(3, "Sohan", [35, 40, 38])

students = [s1, s2, s3]

for student in students:
    student.show_report()
