2. What is a DataFrame?

A DataFrame is the most important object in pandas. It is a table-like structure with rows and columns.

You can think of a DataFrame as an Excel sheet inside Python. Each column has a name. Each row has an index number.

Example table

Index name hours attendance marks
0 Amit 1 55 35
1 Ravi 2 60 42
2 Sita 3 65 50

In this table:

  • name, hours, attendance and marks are columns.
  • 0, 1, 2 are index values.
  • One full horizontal line is called a row.
  • One vertical field is called a column.

Creating a DataFrame

import pandas as pd

data = {
    "name": ["Amit", "Ravi", "Sita"],
    "hours": [1, 2, 3],
    "attendance": [55, 60, 65],
    "marks": [35, 42, 50]
}

df = pd.DataFrame(data)

print(df)

Output

   name  hours  attendance  marks
0  Amit      1          55     35
1  Ravi      2          60     42
2  Sita      3          65     50

Understanding df

df is only a variable name. It is short for DataFrame. We could also write: 

students = pd.DataFrame(data)

But most programmers use df because it is short and common.

Important: pd.DataFrame(data) converts dictionary data into a pandas table. The keys of the dictionary become column names. The list values become column data.

DataFrame, Series and value

Item Meaning Example
DataFrame A full table with rows and columns. df
Series One column from a DataFrame. df["marks"]
Value One single cell. Marks of Amit = 35

Single column gives a Series

marks_column = df["marks"]

print(marks_column)

This gives one column. In pandas, one column is called a Series.

Multiple columns give a DataFrame

selected_columns = df[["hours", "marks"]]

print(selected_columns)

Double square brackets return a DataFrame because we are selecting a list of columns.

Very important for machine learning: For input X, use double square brackets: df[["hours"]]. For output y, single square brackets are usually fine: df["marks"].

3. Every pandas Feature Used in This Lesson

Before we build the full pipeline, we must understand the pandas commands that appear again and again.

3.1 df.head()

print(df.head())

df.head() shows the first 5 rows of the DataFrame. It is used immediately after loading data so that we can check whether the data has loaded properly. 

print(df.head(10))

This shows the first 10 rows.

When to use: Use df.head() after data pulling. It quickly tells you whether the columns and first few records look correct.

3.2 df.tail()

print(df.tail())

df.tail() shows the last 5 rows of the DataFrame. This is useful because sometimes the top rows look fine but the bottom rows may contain empty lines, totals, notes or unwanted data. 

print(df.tail(3))

This shows the last 3 rows.

Example use: If a CSV file was exported from Excel, the last row may contain "Total" or "Generated by software". df.tail() helps catch that.

3.3 df.info()

print(df.info())

df.info() gives technical information about the DataFrame. It tells us:

  • How many rows are present.
  • How many columns are present.
  • Column names.
  • How many non-empty values each column has.
  • Data type of each column.
  • Memory usage.

Example output

<class 'pandas.core.frame.DataFrame'>
RangeIndex: 8 entries, 0 to 7
Data columns (total 4 columns):
 #   Column      Non-Null Count  Dtype
---  ------      --------------  -----
 0   name        8 non-null      object
 1   hours       8 non-null      int64
 2   attendance  8 non-null      int64
 3   marks       8 non-null      int64

How to read this output

Part Meaning
RangeIndex: 8 entries The DataFrame has 8 rows.
Data columns (total 4 columns) The DataFrame has 4 columns.
Non-Null Count How many non-missing values are present.
Dtype The data type of the column.
object Usually text/string data.
int64 Integer number data.
float64 Decimal number data.
Important: If a numeric column appears as object, it may mean the numbers are stored as text. This can create problems during training.

3.4 df.describe()

print(df.describe())

df.describe() gives statistical summary of numeric columns. It is useful for understanding the spread and range of data.

Statistic Meaning
count Number of non-empty values.
mean Average value.
std Standard deviation. It shows how spread out the values are.
min Smallest value.
25% First quartile. 25% of values are below this.
50% Median. Half the values are below this.
75% Third quartile. 75% of values are below this.
max Largest value.
When to use: Use df.describe() before training. If minimum, maximum or average values look strange, clean the data first.

3.5 df.shape

print(df.shape)

df.shape tells the size of the DataFrame. It returns two numbers: 

(rows, columns)

Example: (8, 4) means 8 rows and 4 columns.

3.6 df.columns

print(df.columns)

df.columns shows all column names. It is useful when we are not sure about exact spelling of column names.

3.7 df.dtypes

print(df.dtypes)

df.dtypes shows the data type of each column. This is a shorter version of the data type information shown by df.info().

3.8 df.isnull().sum()

print(df.isnull().sum())

This command checks missing values column by column.

  • df.isnull() checks every cell and returns True if the value is missing.
  • .sum() counts how many missing values are present in each column.

3.9 df.dropna()

df = df.dropna()

dropna() removes rows that contain missing values. This is simple but sometimes dangerous because useful data may be removed.

3.10 df.fillna()

df["attendance"] = df["attendance"].fillna(df["attendance"].mean())

fillna() fills missing values. In this example, missing attendance values are filled with the average attendance.

3.11 df.drop_duplicates()

df = df.drop_duplicates()

This removes duplicate rows. Duplicate rows can make the model biased because the same record gets counted more than once.

3.12 df.rename()

df = df.rename(columns={
    "study hours": "hours",
    "student marks": "marks"
})

This changes column names. Clean column names make code easier to write.

3.13 astype()

df["hours"] = df["hours"].astype(float)
df["marks"] = df["marks"].astype(int)

astype() changes the data type of a column. Machine learning models need numeric columns to be actual numbers, not text.

3.14 sort_values()

df = df.sort_values(by="marks", ascending=False)

This sorts the DataFrame using one column. ascending=False means largest value first.

3.15 groupby()

city_average = df.groupby("city")["marks"].mean()

groupby() groups rows by a column and then applies a calculation. This example finds average marks city by city.

3.16 to_csv(), to_excel(), to_json()

df.to_csv("final_result.csv", index=False)
df.to_excel("final_result.xlsx", index=False)
df.to_json("final_result.json", orient="records", indent=4)
Command Use
to_csv() Saves data as CSV.
to_excel() Saves data as Excel file.
to_json() Saves data as JSON.
index=False Prevents pandas from saving the index as an extra column.

4. Data Pulling

Data pulling means bringing data into your Python program. Data may come from a dictionary, CSV file, Excel file, JSON file, database or online source.

4.1 Pull data from a Python dictionary

import pandas as pd

data = {
    "name": ["Amit", "Ravi", "Sita", "Meena", "Kabir", "Anu", "Farhan", "Pooja"],
    "hours": [1, 2, 3, 4, 5, 6, 7, 8],
    "attendance": [55, 60, 65, 70, 78, 85, 90, 95],
    "marks": [35, 42, 50, 58, 70, 80, 88, 96]
}

df = pd.DataFrame(data)

print(df.head())

Explanation

  • data is a Python dictionary.
  • Each key becomes a column name.
  • Each list becomes values of that column.
  • pd.DataFrame(data) creates the DataFrame.
  • df.head() checks the first few rows.

4.2 Pull data from CSV

df = pd.read_csv("students.csv")

print(df.head())
print(df.info())

read_csv() reads a CSV file and returns a DataFrame.

4.3 Pull data from Excel

df = pd.read_excel("students.xlsx")

print(df.head())

read_excel() reads an Excel file.

4.4 Pull data from JSON

df = pd.read_json("students.json")

print(df.head())

read_json() reads JSON data and converts it into a DataFrame.

5. Data Cleaning

Raw data is often imperfect. Data cleaning means making the data usable.

5.1 Inspect before cleaning

print(df.head())
print(df.tail())
print(df.info())
print(df.describe())
print(df.isnull().sum())

This group of commands is the basic health check of a DataFrame.

Command What it tells us
df.head() Are the first rows correct?
df.tail() Are the last rows correct?
df.info() Are data types and non-null counts correct?
df.describe() Do numeric values look reasonable?
df.isnull().sum() How many missing values are there?

5.2 Remove duplicate rows

df = df.drop_duplicates()

This removes rows that are exactly repeated.

5.3 Fill missing attendance

df["attendance"] = df["attendance"].fillna(df["attendance"].mean())

If attendance is missing, this fills it with the average attendance.

5.4 Remove remaining missing values

df = df.dropna()

This removes rows where any missing value still remains.

5.5 Convert columns to numbers

df["hours"] = df["hours"].astype(float)
df["attendance"] = df["attendance"].astype(float)
df["marks"] = df["marks"].astype(float)

This ensures that the model receives numeric data.

6. Data Querying

Data querying means asking questions from the DataFrame. Before training, we should understand the data.

6.1 Select one column

print(df["marks"])

This returns one column as a Series.

6.2 Select multiple columns

print(df[["hours", "attendance", "marks"]])

This returns a smaller DataFrame with selected columns.

6.3 Filter rows

high_marks = df[df["marks"] >= 70]

print(high_marks)

This selects only those rows where marks are 70 or more.

6.4 Multiple conditions

good_students = df[
    (df["hours"] >= 4) & (df["attendance"] >= 75)
]

print(good_students)

This selects students who studied at least 4 hours and had attendance of at least 75.

Important: In pandas filtering, use & for AND and wrap each condition in brackets.

6.5 Sorting

sorted_df = df.sort_values(by="marks", ascending=False)

print(sorted_df)

This shows the highest scoring students first.

7. Training

Training means teaching a model to learn the relationship between input columns and the target column.

7.1 Choose input and output

X = df[["hours", "attendance"]]
y = df["marks"]
Variable Meaning
X Input data. These are the columns given to the model.
y Target data. This is the answer the model should learn to predict.

7.2 Split into training and testing data

X_train, X_test, y_train, y_test = train_test_split(
    X,
    y,
    test_size=0.25,
    random_state=42
)

train_test_split() divides data into two parts:

  • Training data: used to teach the model.
  • Testing data: used to check the model.

test_size=0.25 means 25% data is used for testing. random_state=42 makes the split repeatable.

7.3 Create model

model = LinearRegression()

LinearRegression() creates a model that tries to learn a straight-line relationship between input and output.

7.4 Train model

model.fit(X_train, y_train)

fit() trains the model. It means: learn from X_train and y_train.

7.5 Test model

score = model.score(X_test, y_test)

print("Model score:", score)

score() checks how well the model performs on test data. For Linear Regression, this is usually the R² score.

7.6 Predict new result

new_student = [[7, 92]]

prediction = model.predict(new_student)

print("Predicted marks:", prediction[0])

predict() asks the trained model to make a prediction.

Why [[7, 92]]? The model expects input as a table. One row with two columns is written as [[7, 92]].

8. Presentation

Presentation means showing the result in a clear and useful way.

8.1 Add prediction column

df["predicted_marks"] = model.predict(X)

print(df)

This creates a new column in the DataFrame. Now the table contains both actual marks and predicted marks.

8.2 Plot actual marks

plt.scatter(df["hours"], df["marks"])
plt.xlabel("Study Hours")
plt.ylabel("Marks")
plt.title("Study Hours vs Marks")
plt.show()

scatter() makes a dot chart. It helps us see the relationship between study hours and marks.

8.3 Plot actual and predicted marks

plt.scatter(df["hours"], df["marks"], label="Actual Marks")
plt.plot(df["hours"], df["predicted_marks"], label="Predicted Marks")

plt.xlabel("Study Hours")
plt.ylabel("Marks")
plt.title("Actual vs Predicted Marks")
plt.legend()
plt.show()
Feature Meaning
plt.scatter() Draws points.
plt.plot() Draws a line.
plt.xlabel() Sets the x-axis label.
plt.ylabel() Sets the y-axis label.
plt.title() Sets chart title.
plt.legend() Shows labels for chart items.
plt.show() Displays the chart.

9. Data Saving

Saving is important because we do not want to repeat all work again. We should save cleaned data, final prediction data and the trained model.

9.1 Save cleaned or final data as CSV

df.to_csv("final_student_result.csv", index=False)

to_csv() saves the DataFrame as a CSV file. index=False prevents pandas from adding index as an extra column.

9.2 Save as JSON

df.to_json("final_student_result.json", orient="records", indent=4)

orient="records" saves each row as one JSON object. indent=4 makes the file readable.

9.3 Save trained model

joblib.dump(model, "student_marks_model.pkl")

joblib.dump() saves the trained model. The saved file can be loaded later without retraining.

9.4 Load trained model

loaded_model = joblib.load("student_marks_model.pkl")

prediction = loaded_model.predict([[7, 92]])

print(prediction[0])

10. Complete Mini Project

This project uses every important idea from the lesson. 

import pandas as pd
import matplotlib.pyplot as plt
import joblib

from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression

# -------------------------------------------------
# 1. DATA PULLING
# -------------------------------------------------

data = {
    "name": ["Amit", "Ravi", "Sita", "Meena", "Kabir", "Anu", "Farhan", "Pooja"],
    "hours": [1, 2, 3, 4, 5, 6, 7, 8],
    "attendance": [55, 60, 65, 70, 78, 85, 90, 95],
    "marks": [35, 42, 50, 58, 70, 80, 88, 96]
}

df = pd.DataFrame(data)

# -------------------------------------------------
# 2. UNDERSTANDING THE DATAFRAME
# -------------------------------------------------

print("First five rows:")
print(df.head())

print("Last five rows:")
print(df.tail())

print("DataFrame information:")
print(df.info())

print("Statistical summary:")
print(df.describe())

print("Shape of DataFrame:")
print(df.shape)

print("Column names:")
print(df.columns)

print("Data types:")
print(df.dtypes)

# -------------------------------------------------
# 3. DATA CLEANING
# -------------------------------------------------

print("Missing values:")
print(df.isnull().sum())

df = df.drop_duplicates()

df["hours"] = df["hours"].astype(float)
df["attendance"] = df["attendance"].astype(float)
df["marks"] = df["marks"].astype(float)

# -------------------------------------------------
# 4. DATA QUERYING
# -------------------------------------------------

print("Students who scored 70 or more:")
print(df[df["marks"] >= 70])

print("Average marks:")
print(df["marks"].mean())

print("Data sorted by marks:")
print(df.sort_values(by="marks", ascending=False))

# -------------------------------------------------
# 5. TRAINING
# -------------------------------------------------

X = df[["hours", "attendance"]]
y = df["marks"]

X_train, X_test, y_train, y_test = train_test_split(
    X,
    y,
    test_size=0.25,
    random_state=42
)

model = LinearRegression()

model.fit(X_train, y_train)

score = model.score(X_test, y_test)

print("Model score:", score)

# -------------------------------------------------
# 6. PRESENTATION
# -------------------------------------------------

df["predicted_marks"] = model.predict(X)

print("Final DataFrame with Predictions:")
print(df)

plt.scatter(df["hours"], df["marks"], label="Actual Marks")
plt.plot(df["hours"], df["predicted_marks"], label="Predicted Marks")

plt.xlabel("Study Hours")
plt.ylabel("Marks")
plt.title("Actual Marks vs Predicted Marks")
plt.legend()
plt.show()

# -------------------------------------------------
# 7. DATA SAVING
# -------------------------------------------------

df.to_csv("final_student_result.csv", index=False)
df.to_json("final_student_result.json", orient="records", indent=4)

joblib.dump(model, "student_marks_model.pkl")

print("Data and model saved successfully.")
