Important note for students

In Java SE 25, the official Java Language Specification lists 51 reserved keywords and 17 contextual keywords. Reserved keywords cannot be used as names. Contextual keywords act like keywords only in special places. true, false, and null are important literals, but they are not technically keywords.

Source note: based on Java Language Specification, Java SE 25, Section 3.9 Keywords.

What is a keyword?

A keyword is a word that Java has already reserved for a special job. You cannot use a reserved keyword as a class name, variable name, or method name.

Wrong

int class = 9;  // error because class is a keyword

Correct

int classNumber = 9;
System.out.println(classNumber);

01

_

Reserved Special identifier

The underscore is a keyword. It cannot be used as a normal one-character variable name. In modern Java it can be used as an unnamed variable in special places.

Example 1: Wrong: underscore as normal variable

// Wrong in Java:
// int _ = 10;

int ignoredValue = 10;

Example 2: Unnamed variable idea in a loop

// Modern Java idea:
// for (String _ : names) {
//     count++;
// }

for (String name : names) {
    count++;
}

        System.out.println("Class 9 result printed");
