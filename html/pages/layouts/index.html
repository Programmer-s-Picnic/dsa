Lesson Goal

Layout means arranging parts of a webpage so the page is readable, balanced, and useful. Good layout decides where the header goes, where the content sits, how cards line up, how a sidebar behaves, and what happens on small screens.

Core principle

  • Use Flexbox for one-direction flow
  • Use Grid for rows and columns together
  • Use both when building real pages

What you should do while learning

  • Copy each example into the editor
  • Change values like gap, columns, and alignment
  • Resize the screen often
  • Observe what changes and why

Level 0 — Layout before Flexbox and Grid

Understand the default behavior first.

A webpage is made of boxes. By default, many elements stack vertically. This is called normal block flow. Understanding this is important because Flexbox and Grid do not replace HTML structure. They organize that structure.

The important child property

flex: 1 tells a child to grow and take available space. This is one of the most useful flex patterns.

Level 3 Code — Sidebar Layout 
<div class="layout">
  <aside class="sidebar">Sidebar</aside>
  <main class="main">Main content goes here.</main>
</div>

<style>
  .layout{
    display:flex;
    gap:16px;
  }

  .sidebar{
    flex:0 0 220px;
    padding:20px;
    background:#fff2dd;
    border-radius:16px;
  }

  .main{
    flex:1;
    padding:20px;
    background:#fffdf9;
    border-radius:16px;
  }

  @media (max-width: 800px){
    .layout{
      flex-direction:column;
    }

    .sidebar{
      flex-basis:auto;
    }
  }
</style>

<div class="app">
  <header class="app-header">
    <div>Logo</div>
    <nav class="menu">
      <a href="#">Home</a>
      <a href="#">Reports</a>
      <a href="#">Settings</a>
    </nav>
  </header>

  <aside class="left">Left Sidebar</aside>
  <main class="main">Main Content</main>
  <aside class="right">Right Panel</aside>
  <footer class="foot">Footer</footer>
</div>

<style>
  .app{
    display:grid;
    grid-template-columns:220px 1fr 220px;
    grid-template-areas:
      "header header header"
      "left main right"
      "foot foot foot";
    gap:16px;
  }

  .app-header{
    grid-area:header;
    display:flex;
    justify-content:space-between;
    align-items:center;
    gap:14px;
    padding:18px;
    background:#fff2dd;
    border-radius:16px;
  }

  .menu{
    display:flex;
    gap:14px;
    flex-wrap:wrap;
  }

  .left{ grid-area:left; }
  .main{ grid-area:main; }
  .right{ grid-area:right; }
  .foot{ grid-area:foot; }

  .left, .main, .right, .foot{
    padding:18px;
    background:#fffdf9;
    border-radius:16px;
    border:1px solid #ead7c4;
  }

  @media (max-width: 900px){
    .app{
      grid-template-columns:1fr;
      grid-template-areas:
        "header"
        "left"
        "main"
        "right"
        "foot";
    }

    .app-header{
      flex-direction:column;
      align-items:flex-start;
    }
  }
</style>

Subgrid — what it is and why it matters

Subgrid is a Grid feature that lets a child grid align itself to the track sizes of its parent grid. In simpler words, it helps nested grid sections line up with the parent structure instead of creating an unrelated internal grid.

Simple idea

Without subgrid, nested cards may align internally but not line up nicely with the outer page structure. With subgrid, children can inherit row or column tracks from the parent grid.

Subgrid Example 
.parent{
  display:grid;
  grid-template-columns:200px 1fr 1fr;
  gap:16px;
}

.child{
  display:grid;
  grid-column:2 / 4;
  grid-template-columns:subgrid;
}

Important note

You do not need subgrid to become good at layout. It is useful later, especially for polished editorial or dashboard designs.

Final Summary

The learning path is now complete:

  1. Understand normal document flow.
  2. Learn Flexbox for one-direction layouts.
  3. Learn alignment, wrapping, and growing items.
  4. Use Grid for rows and columns together.
  5. Use responsive grid functions for real layouts.
  6. Combine Grid and Flexbox for dashboards and full pages.

Golden rule

Use Flexbox when content mainly flows in one direction. Use Grid when structure matters in both directions.

Best next projects

Build a navbar, a responsive card gallery, and then a dashboard. That sequence will make the ideas stick.
