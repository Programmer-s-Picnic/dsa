Multi-Level Layout Lesson

Learn Multi-Level Layouts by Building a Visit Varanasi Page

In this lesson, we will not learn layout in an abstract way only. We will use a real topic: a travel page for Varanasi. Step by step, we will build outer layout, nested hero layout, attraction cards, travel planning sections, and a sidebar-based final travel page.

Lesson Goal

A multi-level layout means a layout inside another layout. For example:

Main idea

  • The whole page may use Grid.
  • The hero inside it may also use Grid.
  • The toolbar inside a section may use Flexbox.
  • The attraction cards inside the content may use Grid again.

Travel page reason

A travel page is a very good practice project because it naturally has many layout levels: hero, highlights, attraction cards, itinerary, travel notes, and sidebar information.

Level 1 — Build the hero using nested Grid

The hero is our first multi-level layout.

The outer hero section uses Grid with two columns: the left side for title and text, and the right side for a travel summary card. Inside the left side, we stack smaller pieces vertically.
Visit Varanasi hero layout
Small Eyebrow + Heading
Short Intro Text
Buttons Row
Side Travel Card
Best Time • Ghats • Temples • Food

Why Grid here?

Because we want two main zones together: a big content area and a side card. That is a two-dimensional section layout, so Grid fits very well.

Hero layout code 
.hero-layout{
  display:grid;
  grid-template-columns:1.15fr .85fr;
  gap:16px;
}

  gap:12px;

Layout lesson

A page can contain many different grids. One grid does not control everything. Each section can get its own layout rules depending on its purpose.

Cards grid code 
.quick-info{
  display:grid;
  grid-template-columns:repeat(3, 1fr);
  gap:16px;
}

.attractions-grid{
  display:grid;
  grid-template-columns:repeat(3, 1fr);
  gap:16px;
}

@media (max-width: 900px){
  .quick-info,
  .attractions-grid{
    grid-template-columns:1fr 1fr;
  }
}

@media (max-width: 640px){
  .quick-info,
  .attractions-grid{
    grid-template-columns:1fr;
  }
}

Level 3 — Build a trip plan with main area + side info

This section shows another layout inside the same page.

The trip planner is a very natural two-column layout: the left side can show day-wise itinerary, and the right side can show quick travel notes, weather reminders, or transport tips.
Trip planner layout
Trip Planner Heading
Day 1 — Ghats + Temple
Day 2 — Sarnath + BHU
Day 3 — Food + Walks
Day 4 — Shopping + Sunrise Boat Ride
Travel Tips
Best Time to Visit
What to Carry

Why not use one long column?

Because the user can understand the travel plan faster when the main content and supporting notes are visually separated.

Final Summary

You used a real “Visit Varanasi” page to understand multi-level layouts.

What you learned

  1. Think in large page sections first.
  2. Use Grid for outer and section-level structure.
  3. Use Flexbox for smaller horizontal control rows.
  4. Nest layouts inside layouts for real-world pages.
  5. Travel pages are excellent practice for layout mastery.

Best next step

Convert this lesson into a real tourism page with images, maps, day-wise itinerary, local food guide, and booking/contact sections.
