0. Level 0: What is a CSS selector?

A CSS selector tells the browser which HTML element should receive a style. At the beginner level, we usually select normal HTML tags, classes, or IDs. 

<h1>Programmers Picnic</h1>
<p>Learn CSS step by step.</p>
h1 {
  color: orange;
}

p {
  color: #333;
}
Here, h1 selects all heading elements and p selects all paragraph elements.

1. What is :root?

:root means the highest-level parent of the page. In an HTML document, :root usually represents the <html> element. 

:root {
  --main-color: #d97706;
  --bg-color: #fff7ed;
  --text-color: #241407;
}

The most common use of :root is to store CSS variables. These variables can control the whole page theme from one place.

2. CSS Variables

CSS variables are reusable values. They usually start with two hyphens. 

:root {
  --saffron: #f59e0b;
  --dark: #1f2937;
  --light: #fff7ed;
}

Now we can use these values anywhere with var().

body {
  background: var(--light);
  color: var(--dark);
}

button {
  background: var(--saffron);
}

Without variables

You repeat the same color again and again. Changing the design becomes slow.

With variables

You change the value once inside :root and the whole design updates.

4. What is ::before?

::before creates a fake element before the real content of an element. The content is not written in HTML. CSS adds it visually. 

<h2>CSS Lesson</h2>
h2::before {
  content: "★ ";
}

5. What is ::after?

::after creates a fake element after the real content of an element. 

h2::after {
  content: " ✓";
}

Live Demo

CSS Lesson ✓

Important Rule

For ::before and ::after, the content property is usually required. 

.box::after {
  content: "";
  display: block;
  width: 100px;
  height: 4px;
  background: orange;
}

6. Decoration With ::before and ::after

Pseudo-elements are often used for decoration, badges, shapes, icons, and underlines. 

.title {
  position: relative;
  display: inline-block;
}

.title::after {
  content: "";
  position: absolute;
  left: 0;
  bottom: -8px;
  width: 100%;
  height: 5px;
  background: orange;

7. Single Colon vs Double Colon

Type Examples Meaning
Pseudo-class :hover, :focus, :root, :first-child Selects a state or condition.
Pseudo-element ::before, ::after, ::selection, ::first-letter Creates or styles a special part of an element. 
/* Modern pseudo-element style */
h1::before {
  content: "Modern ";
}

/* Old style still often works */
h1:before {
  content: "Old ";
}
Use double colon for pseudo-elements. Use single colon for pseudo-classes.

8. Pseudo-class vs Pseudo-element

Pseudo-class

A pseudo-class selects a state or condition.

button:hover {
  background: black;
}

Pseudo-element

A pseudo-element creates or styles a part of an element.

button::after {
  content: " →";
}

Common Pseudo-classes

.card:hover {
  transform: translateY(-5px);
}

input:focus {
  border-color: orange;
  outline: none;
}

li:first-child {
  font-weight: bold;
}

li:last-child {
  color: red;
}

li:nth-child(odd) {
  background: #fff7ed;
}

10. Practical Patterns

Pattern 1: Heading underline

.heading::after {
  content: "";
  display: block;
  width: 80px;
  height: 4px;
  background: orange;
  margin-top: 10px;
}

Pattern 2: Required field star

label.required::after {
  content: " *";
  color: red;
}

Pattern 3: Quote mark

.quote::before {
  content: "“";
  font-size: 60px;
  color: orange;
}
CSS pseudo-elements help us add visual meaning without adding extra HTML.

Pattern 4: External link arrow

a.external::after {
  content: " ↗";
}

Pattern 5: List check icon

li::before {
  content: "✓";
  color: green;
}

::before as a background layer

  position: relative;
  overflow: hidden;
}

.hero::before {
  content: "";
  position: absolute;
  inset: 0;

  z-index: -1;

::after as tooltip

Hover on me

  position: relative;

  position: absolute;

  bottom: 130%;

  opacity: 0;
  pointer-events: none;

}

13. Common Mistakes

Mistake 1: Forgetting content

/* Wrong */
.box::after {
  background: red;
}

/* Correct */
.box::after {
  content: "";
  background: red;
}

Mistake 2: Forgetting display

.box::after {
  content: "";
  display: block;
  width: 100px;
  height: 5px;
  background: orange;
}

Mistake 3: Forgetting position: relative

.card {
  position: relative;
}

.card::before {
  content: "";
  position: absolute;
}

Mistake 4: Using pseudo-elements directly on some replaced elements

Pseudo-elements may not work reliably on elements like <img>, <input>, and <br>. Wrap them in another element. 

<div class="image-wrap">
  <img src="photo.jpg" alt="Photo">
</div>

.image-wrap::after {
  content: "";
}

16. Final Memory Map

:root Stores global CSS variables.
:hover Styles an element when the mouse is over it.
:focus Styles an input or button when it is selected.
:first-child Selects the first child.
:last-child Selects the last child.
:nth-child() Selects elements by position or pattern.
::before Creates fake content before real content.
::after Creates fake content after real content.
::selection Styles selected text.
::first-letter Styles the first letter.
::first-line Styles the first visible line.
Main idea: use :root for the design system, use ::before and ::after for clean decorative elements, and use pseudo-classes for states and conditions.
