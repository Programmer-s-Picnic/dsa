1. What is Animation?

Animation means changing something slowly over time so that the user sees movement.

Without animation

A button suddenly changes color. A box suddenly appears. A menu suddenly opens.

With animation

The color smoothly changes. The box fades in. The menu slides open.

Why use animation?

Animation helps users understand movement, attention, loading, success, error, and interaction.

Checkpoint: Animation is not only decoration. Good animation explains what is happening.

The three main web animation tools

CSS Transition

Best for simple hover, focus, open, close, and state changes.

CSS Keyframes

Best for repeating animations like bouncing, spinning, loading, and moving objects.

JavaScript

Best when the user controls animation through buttons, sliders, data, or logic.

2. CSS Transition: The First Animation

A transition makes a change smooth. Move your mouse over the card below.

Hover me 
.card {
  transition: transform 0.5s ease, background 0.5s ease;
}

.card:hover {
  transform: scale(1.1) rotate(4deg);
  background: purple;
}

Meaning of the code

transition

Tells the browser which property should change smoothly.

0.5s

The animation duration. Here it takes half a second.

ease

The speed style. It starts slowly, speeds up, then slows down.

Checkpoint: Transition needs two states: a starting state and an ending state.

3. CSS Keyframes: Repeated Animation

@keyframes lets us describe animation steps. Below, the ball moves from one vertical position to another.

CSS
@keyframes bounce {
  from {
    transform: translateY(60px);
  }

  to {
    transform: translateY(-60px);
  }
}

.ball {
  animation: bounce 1s ease-in-out infinite alternate;
}

Important animation words

animation-name

Name of the keyframe animation.

animation-duration

How long one cycle takes.

animation-iteration-count

How many times it repeats. infinite means forever.

animation-direction

alternate means go forward, then backward.

Checkpoint: Use transition for simple state change. Use keyframes for planned motion.

4. Transform: Move, Rotate, Scale, Skew

Most modern animations use transform. It is fast and smooth because browsers optimize it well. 

transform: translateX(50px);
transform: translateY(50px);
transform: rotate(45deg);
transform: scale(1.5);
transform: skew(10deg);

translate

Moves an element.

rotate

Turns an element.

scale

Makes an element bigger or smaller.

skew

Tilts an element.

Important: Prefer animating transform and opacity. Avoid heavy animation of width, height, top, and left when possible.

5. Timing Functions: The Feeling of Motion

Animation is not only about movement. It is also about speed. A robot-like movement feels different from a natural movement.

Move
element.style.transition = "transform 1s ease";
element.style.transform = "translateX(220px)";
Checkpoint: The same distance can feel different when the timing function changes.

9. Scroll Animation

Scroll animation means an element appears when it enters the screen. Scroll down and watch these cards appear.

Step 1

The element starts hidden using opacity 0.

Step 2

JavaScript observes when the element enters the screen.

Step 3

A class is added. The element fades and moves into place.

const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add("visible");
    }
  });
});

document.querySelectorAll(".scroll-card").forEach(card => {
  observer.observe(card);
});

10. Animation Properties Cheat Sheet

animation-name

Name of the keyframes.

animation-duration

Time taken by one cycle.

animation-timing-function

Speed curve: linear, ease, ease-in, ease-out.

animation-delay

Wait before starting.

animation-iteration-count

Number of repeats.

animation-direction

normal, reverse, alternate, alternate-reverse.

animation-fill-mode

Controls style before and after animation.

animation-play-state

running or paused.

13. Full Mini Project: Animated Button

Here is a small complete animation project. Copy it into a new HTML file and run it. 

<!doctype html>
<html>
<head>
  <style>
    body {
      height: 100vh;
      display: grid;
      place-items: center;
      background: #fff7ed;
      font-family: Arial, sans-serif;
    }

    button {
      padding: 18px 30px;
      border: none;
      border-radius: 999px;
      background: #ff8a00;
      color: white;
      font-size: 22px;
      font-weight: bold;
      cursor: pointer;
      transition: transform 0.3s ease, box-shadow 0.3s ease;
      animation: softPulse 1.5s ease-in-out infinite alternate;
    }

    button:hover {
      transform: scale(1.12) rotate(-2deg);
      box-shadow: 0 16px 30px rgba(249, 115, 22, 0.35);
    }

    @keyframes softPulse {
      from {
        box-shadow: 0 0 0 rgba(249, 115, 22, 0.2);
      }

      to {
        box-shadow: 0 0 35px rgba(249, 115, 22, 0.65);
      }
    }
  </style>
</head>

<body>
  <button>Start Learning Animation</button>
</body>
</html>

14. Final Understanding

Animation begins with a simple idea: change something over time. In web development, the most important tools are:

Transition

For smooth change between two states.

Keyframes

For planned motion with steps.

Transform

For fast movement, rotation, and scaling.

JavaScript

For user-controlled animation.

Infinity Rule: Do not animate everything. Animate only when it helps the user understand, enjoy, or interact.
